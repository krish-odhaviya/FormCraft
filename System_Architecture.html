System Overview

Frontend (Client)

Built with Next.js 15+ (App Router). Handles form construction, dynamic rendering of published forms, and administrative dashboards.

Backend (API)

Powered by Spring Boot 3.5+. Manages business logic, dynamic schema generation, and session-based authentication.

Database

Utilizes PostgreSQL. Features a hybrid design with static metadata tables and dynamic transaction tables for form submissions.

Security Architecture

FormCraft implements a multi-layered security strategy following OWASP best practices.

Authentication

Session-based authentication using Spring Security 6. Credentials hashed via BCrypt. Concurrent session control limited to 1 session per user.

CSRF Protection

Enabled for all mutating requests. Uses XSRF-TOKEN cookie-based mechanism compatible with SPA frameworks like Next.js.

Dynamic Data Security

SQL Injection prevented via sanitized identifiers (SchemaService) and parameterized queries (JdbcTemplate) for dynamic table operations.

Technical Deep Dives

1. Dynamic Schema Engine (SchemaService)

CORE LOGIC

How FormCraft converts UI layouts into physical PostgreSQL architecture without performance overhead.

// The Materialization Logic

  1. Sanitization: Converts "Full Name ($)" to full_name_.
  2. Collision Check: If table form_data_user_feedback exists, appends unique UUID suffix.
  3. DDL Generation: Constructs CREATE TABLE with system columns (id, version_id, created_at).
  4. Migration: If form is updated, executes ALTER TABLE ADD COLUMN. It never removes columns to prevent data loss.

2. Dynamic Submission Engine (FormSubmissionService)

DATA FLOW

Managing arbitrary JSON payloads into structured SQL columns with rigorous validation.

A. Validation Phase

Checks system-level constraints (Required, Pattern, Min/Max) and Cross-field rules using a custom Logic Evaluator before any DB hit.

B. Injection via JdbcTemplate

Dynamically builds parameterized SQL. Complex fields (multiselect, grids) are cast to ?::jsonb to allow rich querying.

3. Cross-Field Logic Engine (RuleEngineService)

BUSINESS LOGIC

The "Brain" that handles visibility (show/hide) and complex validation expressions.

4. Version Control System (FormVersionService)

GOVERNANCE

Implements the "Stable Runtime" principle. Public users never see breaking changes until explicit activation.

  • Draft Working Copy: Stored purely as JSON in form_version. No DB tables are affected.
  • Active Snapshot: The immutable state used for public rendering and data capture.
  • Archival: Historical versions are retained allowing system-wide audits and data restoration.
